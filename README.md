`gum` is a markup language for configuring software inspired by `TOML`, `YAML`, `Nix`, and `JSON`. Take a look at the [specification](./SPEC..md) to see how it works, or head over the the [website]() for broader documentation.

There are a number of standardized [packages]() for whatever language you're into at the moment:

- [Python](https://github.com/gum-lang/python)
- ~~[TypeScript]()~~
- ~~[Java]()~~
- ~~[Kotlin]()~~
- ~~[Go]()~~
- ~~[Rust]()~~
- ~~[Odin]()~~
- ~~[C]()~~
- ~~[Zig]()~~

There is also a [CLI]() for validation, formatting, and conversion, and an [LSP]() with several editor plugins available:

- ~~[VSCode]()~~
- ~~[Neovim]()~~
- ~~[Zed]()~~
